# Бугаков Электромонтаж — сайт-визитка

Одностраничный адаптивный сайт частного мастера-электрика (Омск).
Чистый HTML / CSS / JS, без сборки и зависимостей — открывается как есть.

## Структура проекта

```text
bugakov-elektro-site/
├── index.html              # вся страница
├── styles.css              # стили (тёмная тема, жёлто-оранжевые акценты)
├── script.js               # меню, анимации, галерея, эффект схемы
├── data/
│   ├── gallery.json        # список фото работ
│   └── before-after.json   # пары «до / после»
└── images/
    ├── master/
    │   └── vyacheslav-avatar.png   # фото мастера (аватарка)
    ├── works/                      # сюда кладём фото работ
    └── before-after/               # сюда кладём пары «до / после»
```

## Как добавить фото работ

1. Положите снимок в папку `images/works/`.
   Имя — латиницей, без пробелов, например: `work-01.jpg`, `shfield-2026.jpg`.
2. Откройте `data/gallery.json` и добавьте запись в массив:

```json
[
  {
    "src": "images/works/work-01.jpg",
    "title": "Сборка электрощита",
    "category": "Щитки"
  },
  {
    "src": "images/works/work-02.jpg",
    "title": "Разводка электрики",
    "category": "Проводка"
  }
]
```

- `src` — путь к файлу (обязательно).
- `title` — подпись под фото.
- `category` — короткая метка категории (необязательно).

3. Сохраните файл и обновите страницу — фото появится.
   Каждую запись отделяйте запятой; у последней записи запятой нет.

---

## Как добавить пару «до / после»

1. Положите два снимка в `images/before-after/`, например `before-01.jpg` и `after-01.jpg`.
2. Добавьте запись в `data/before-after.json`:

```json
[
  {
    "before": "images/before-after/before-01.jpg",
    "after": "images/before-after/after-01.jpg",
    "title": "Замена электрощита"
  }
]
```

3. Сохраните и обновите страницу.

> Совет: фото лучше сжать до ширины ~1200 px и веса до ~300 КБ, чтобы сайт быстро открывался на телефоне.

## Локальный просмотр

Галерея подгружается через `fetch`, поэтому при открытии файла напрямую
(`file://`) фото могут не загрузиться из-за политики браузера. Запустите
простой локальный сервер из папки проекта:

```bash
# Python 3
python -m http.server 8000
# затем откройте http://localhost:8000
```

Текст, услуги, прайс и контакты видны и без сервера.

## Деплой

### GitHub Pages
1. Создайте репозиторий и загрузите туда все файлы.
2. В репозитории: **Settings → Pages**.
3. **Source:** `Deploy from a branch`, ветка `main`, папка `/ (root)`.
4. Сохраните — через минуту сайт будет доступен по адресу
   `https://<ваш-логин>.github.io/<имя-репозитория>/`.

### Netlify
1. Зарегистрируйтесь на [netlify.com](https://netlify.com).
2. **Add new site → Deploy manually** и перетащите папку проекта целиком.
3. Сайт публикуется сразу; позже можно подключить свой домен.

### Свой домен
Домен (например `bugakov-elektro.ru`) подключается в настройках
GitHub Pages или Netlify после покупки — это можно сделать позже.
